# Innerbloom Social Creative System v1

Every asset must be publish-ready, product-led, premium, and designed to acquire users.

The screenshot is evidence, not decoration. The claim and product module must directly match.

For each post define one primary message, funnel role, product-truth anchor, visual proof goal, recommended modules, forbidden modules, and CTA.

Static post: exactly one final asset. Carousel: exactly the number of slides in the approved slide plan. Each slide needs a role such as hook, explanation, mechanism, proof, objection, or CTA.

Use Emotion Chart for emotional patterns; dashboard, tasks, and weekly charts for visible progress; rhythm selection for onboarding and realistic targets; Daily Energy for accumulated momentum; Habit Achieved for earned shareable progress.

Reject an asset when the language is wrong, typography or logo is approximated, the screenshot contradicts the claim, product behaviour is misrepresented, the piece is generic, the hierarchy is unclear, the carousel is incomplete, or text is unreadable.

Before production answer: What is the claim? Which module proves it? Why? Which module would mislead? What must be understood in two seconds? What action follows?
